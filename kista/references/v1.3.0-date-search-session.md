# Kista v1.3.0 Date Search & Auto-Timestamp — Build Session Log

Date: 2026-05-09

## Features Added

### 1. Date/Time Search Flags
- Made `query` argument optional in `kista search`
- Added `--date`, `--from`, `--to`, `--after`, `--before` flags
- All accept `YYYY-MM-DD` format
- Date-only searches work: `kista search --date 2026-05-09`
- Combined: `kista search "github" --from 2026-05-01 --to 2026-05-09`

### 2. Auto-Timestamp on Notes
- `note` entries auto-prepend `[YYYY-MM-DD HH:MM UTC]` on both `add` and `update`
- Format: `[2026-05-09 08:00 UTC] Hello world`
- On update: detects existing timestamp prefix and replaces it with current time
- Timestamp uses `datetime.now(timezone.utc).strftime("[%Y-%m-%d %H:%M UTC]")`

## Test Changes

- All note content assertions updated to use `.endswith()` instead of `==` 
- `assert result["content"].endswith("Hello world")` instead of `assert result["content"] == "Hello world"`
- Update test checks that timestamp is replaced (not doubled) on note update
