## What Changed

Added 7 new entry types beyond the original `credential` type:
- `apikey`: API keys/tokens (key, expires, service_url, scopes, rate_limit)
- `sshkey`: SSH key pairs (private_key via --key-file, public_key, passphrase, key_type, host)
- `certificate`: TLS/SSL certs (cert via --cert-file, key via --key-file, domain, issuer, expires, chain)
- `note`: Secure notes, recovery phrases, seed phrases (content, category)
- `totp`: 2FA/TOTP secrets (secret, digits, period, algorithm, issuer)
- `license`: Software license keys (key, product, seats, expires, order_id)
- `identity`: Personal ID documents (full_name, birth_date, id_number, id_type, address, phone, national_id)

## Design Decisions

1. **Backward compatible** — entries without `entry_type` field default to `credential`. Vault version bumps on first write.
2. **Shortcut commands** — `add-apikey`, `add-note`, etc. auto-set `--type` and provide type-specific flags
3. **Secure file input** — `--key-file` for SSH private keys, `--cert-file` for certificates (never CLI args for large secrets)
4. **Common args on all subcommands** — `--tags` and `--notes` available on all `add-*` shortcuts (initially missing, patched manually)

### Key Pitfalls
- **Forge Worker shortcut subcommands missing common args**: Each `add-*` parser needed `--tags` and `--notes` added individually. The `_add_type_args()` function only added type-specific args, not the common ones.
- **`--expires` dest collision**: Certificate and license types both use `--expires` but for different schema fields. Resolved with `dest="expires_field"` for those types.
- **`--key` dest collision**: API keys and licenses both use `--key` but mean different things. Resolved with `dest="key_arg"`.
- **`--phone` dest collision**: Identity type `--phone` would conflict with general `--phone`. Resolved with `dest="phone_arg"`.
