# 29 — k9s

> https://k9scli.io/

Kubernetes TUI. The reference TUI for "lots of objects in many
namespaces with varied operations on each." Teaches Stofa about the
**command palette as primary navigation**, **resource-as-row**, and
**themability**.

---

## What it is

A Go TUI for managing Kubernetes clusters. Lists pods, services,
deployments, etc., across namespaces. Lets the operator describe,
edit, port-forward, exec into containers — without leaving the TUI.

```
┌── Context: minikube-dev ── Namespace: default ──────────────────┐
│                                                                  │
│ NAME              READY   STATUS    RESTARTS   AGE   IP           │
│ ember-pod-1       1/1     Running   0          2d    10.42.0.13   │
│ ember-pod-2       1/1     Running   0          2d    10.42.0.14   │
│ stofa-test        0/1     Pending   0          5m    -            │
│ ollama-host       1/1     Running   0          7d    10.42.0.5    │
│                                                                  │
│ :pods                                                            │
└──────────────────────────────────────────────────────────────────┘
```

---

## The clever idea: `:` is the primary navigation

In k9s, the operator presses `:` and types a resource name:

- `:pods` → pod list
- `:svc` → services
- `:deploy` → deployments
- `:ns` → namespaces

This is a *command palette as navigation*. Operators don't have to
remember screen-jumping keys; they just type what they want.

### What we steal

Stofa's `:` / `Ctrl-P` command palette is k9s-shaped. Type fuzzy:

- `:theme` → theme picker
- `:chat` → ChatScreen
- `:well` → WellScreen
- `:doctor` → DoctorScreen
- `:plugin` → plugin manager (V2)
- `:export-audit` → export audit log

Every screen + named action is here. The operator who doesn't
remember `c` for chat just types `:chat`.

---

## The theme system

k9s ships ~30 themes. Theme switching is a config-file change +
restart in k9s; we improved on this (live theme swap, no restart).

### What we steal

The *quantity* of themes is aspirational for V2/V3. V1 ships 5
built-in; plugin themes add more. Live-swap is a Stofa improvement
over k9s.

### What we avoid

The "anyone can ship a theme" without validation. k9s themes can
override anything; some are unreadable. Stofa's 20-token contract
([`../architecture/15_THEMING_SYSTEM.md`](../architecture/15_THEMING_SYSTEM.md))
prevents broken themes.

## The resource-as-row pattern

In k9s, each pod is one row. Columns are aligned. Sort by column.
Filter via `/`. Select with j/k. Act with letters.

### What we steal

Stofa's WellScreen, MCPScreen, and (V2) AuditLogScreen all use
this pattern:

- Aligned columns
- `j`/`k` to navigate rows
- `/` to filter
- Letter keys for actions on the selected row

---

## What k9s gets wrong (and we improve)

- **The startup time.** k9s takes a few seconds to connect to a
  cluster and list resources. Stofa launches < 500ms (test in CI).
  Different domain — k8s is over the network — but the lesson is to
  show *something* immediately and populate as data arrives.
- **The reliance on `:` for everything.** In k9s, you almost have to
  use `:`. Stofa gives single-letter jumps too, so the operator can
  press `c` for chat without typing.

---

## What we don't need from k9s

- Kubernetes-specific concepts (namespaces, contexts, RBAC).
- The port-forward / exec-into-pod features.
- Cluster context switching.

---

## Specific Stofa borrowings

- **`:` opens fuzzy command palette** with every named action.
- **`/` opens filter** in list views.
- **Color-coded status** in the leftmost column of every list.
- **Letter keys for row actions** (lowercase = safe, uppercase or
  Shift = destructive with confirm).

---

## Closing

k9s proves that operators happily learn `:` + verb as a primary
navigation idiom. Stofa borrows the pattern and combines it with
single-letter direct jumps, giving operators two ways to navigate:
the typing way (k9s-shaped) and the muscle-memory way (lazygit-shaped).
