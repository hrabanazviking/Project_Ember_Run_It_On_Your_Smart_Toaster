# 99 — Phase 4: The Feast (V3.0+)

The polish + community + "what do operators actually ask for"
phase. Not a fixed scope; a *direction*.

---

## Phase name

**The Feast** — Stofa as a community-shaped, mature platform.

---

## Scope (sketches, not commitments)

V3.0 explores:

1. **Auto-theme-by-clock.** Aurora at night, Midgard during day.
2. **Saved prompt templates.** Operators save common prompts;
   `/template <name>` inserts.
3. **Multi-tab chat.** Multiple concurrent conversations.
4. **HomeScreen customization.** Operator picks panels.
5. **Plugin marketplace UI** (browse + install in Stofa).
6. **`ember plugin install <name>` CLI** wrapper.
7. **Streaming markdown rendering** (live re-render as code blocks
   appear).
8. **Voice input** (when Rödd ships in slice-3).
9. **Bifröst HTTP gateway integration** (Stofa can query other
   Ember instances).
10. **Community theme contests** (twice yearly; winners promoted).

These are sketches. None are committed. The actual V3 scope
depends on what operators ask for in the V2 release window.

---

## Scope refinement principle

Every V3 feature must clear:

1. **At least three operators have asked for it** (in issues,
   Discord, etc.).
2. **A community member has prototyped it as a plugin** (to prove
   the desire is real, not theoretical).
3. **A Mythic-Engineering design pass** has approved the integration.

This is more rigorous than V1/V2's "we think this would be nice"
because by V3 we're shaping the product around real users.

---

## What V3 is NOT

- A redesign. Stofa's bones are V1's. V3 polishes.
- A pivot. Stofa stays a TUI; the GUI is Auga's job.
- An expansion of Ember-the-agent's scope. New agent capabilities
  land in slices (slice-3, slice-4); Stofa surfaces them.

---

## V3 ship rhythm

By V3, Ember has been around for a year+. Ship rhythm shifts:

- **Major versions** (V3.0, V4.0): yearly, with substantial design
  documentation.
- **Minor versions** (V3.1, V3.2): quarterly, with smaller scope.
- **Patches** (V3.0.1): as needed, for bugs.

No fixed cadence; we ship when it's right.

---

## The "decade" view

If Stofa is around in 2036, what does it look like?

Best guess:

- Same TUI shape (terminal UIs have proven durable).
- 5x the pets (community contributions).
- A dozen themes.
- Three or four community-shipped screens.
- Integration with whatever the AI-tool ecosystem becomes.
- Still cozy.

The constraints that keep it cozy across the decade are the
Vows and the Design Philosophy. As long as those hold, Stofa
remains *itself* even as features accrete.

---

## Closing

The Feast is **the long horizon**. We don't plan it in detail
because we don't know what operators will want in two years. We
plan only that we'll *keep listening*, *keep shipping*, *keep
restrained*. The Hearth becomes the Hall, the Hall becomes the
Familiars, the Familiars becomes the Feast — and operators come,
sit, eat, talk, leave content.

The hall, well-built, lasts generations.

That's Stofa.
